Offers to use the schema syntax (since Avro 1.12.0) to replace data-only protocols.

Protocols that have no message declarations (either defined or imported) can be removed in favour of the schema syntax. The schema syntax also has the benefit of defining a main schema, creating a .avsc equivalent.

Example:

@namespace("my.name.space")
protocol MessageLess {
	record Message {
		string title;
		string content = "";
	}

	enum Status {
		ON, OFF, UNKNOWN
	} = UNKNOWN;
}

The quick fix unwraps the protocol body, adds a namespace declaration (when appropriate) and a schema declaration (for the first named schema).

Result after applying the fix: 

namespace my.name.space;
schema Message;

record Message {
	string title;
	string content = "";
}

enum Status {
	ON, OFF, UNKNOWN
} = UNKNOWN;
